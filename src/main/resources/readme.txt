Инструкция:

1. Скопировать на локальный диск.
2. Сконфигурировать файл "property" (см. описание ниже).
3. ВНИМАНИЕ: параметр "withUpdate" в property-файле при первом запуске нужно установить в false. После первого запуска нужно будет установить его в true и больше не менять.
ПРИМЕЧАНИЕ: этот пункт касается случаев, если вы используете БД с любого сервера, скачанную до 22.09.2014. В противном случае, можно сразу ставить withUpdate=true и не заморачиваться (что и рекомендуется: забрать с эталона актуальную БД).
4. Настроить запуск sql-updater через eclipse (см. manual.png).


Описание параметров файла "property" и пример их заполнения:

projectpath=D:\\work\\svnws\\eVorsorge_Eugene - путь к проекту (без слеша в конце!)
tempDirPath=C:\\files - путь к временной папке (она используется в ходе выполнения скриптов), которая стирается (!) после выполнения скрипта (без слеша в конце!)
dbHost=localhost - хост // пока эта настройка не воспринимается
dbName=portal_backup - имя БД
dbPort=3306 - порт MYSQL
dbUser=root - имя пользователя
dbPassword= - пароль пользователя
withUpdate=true - параметр принимает true или false, влияет на то, будут ли выполняться скрипты или только логгироваться.
mysqlPath=C:\\Program Files\\MySQL\\MySQL Server 5.0\\bin - этот параметр необходим в том случае, если MYSQL не прописан в переменных среды. В нем необходимо указать путь в папке bin MYSQL (без слеша в конце!)
